Import and Using

Example 1: Variable of the same name are separated by module

module Bar
    x = 1
    foo() = x
    export foo
end
Main.Bar
julia> import .Bar
julia> foo()ERROR: UndefVarError: `foo` not defined
julia> x = -1;
julia> Bar.foo()1
julia> Bar.x1

Example 2: Redefine/Assign exported global variable?

It is not allowed once x is "called":

module Bar2
    x = 1
    foo() = x
    export foo
    export x # note the difference
end
Main.Bar2
julia> using .Bar2
julia> x # Here is the difference!1
julia> x = -1ERROR: cannot assign a value to imported variable Bar2.x from module Main
julia> foo()1
julia> x === Bar2.xtrue

But it is allowed before x is called

julia> using .Bar2
julia> x = -1-1
julia> foo()1
julia> x === Bar2.xfalse

Difference between using and import

Extensibility of function

  • Function are extensible under the introduced module (e.g., Bar) for both using and import cases.
  • Only explicitly imported function is extensible in the current module (@__MODULE__, e.g., Main).
  • Also see this post.
import case
julia> import .Bar
julia> Bar.foofoo (generic function with 1 method)
julia> Bar.foo(y) = Bar.x + y
julia> Bar.x = 999999
julia> Bar.foo(3)1002
using case
julia> using .Bar2
julia> Bar2.foo(y) = Bar2.x + y; Bar2.x = 999; foo(3)1002
julia> foo(y) = Bar2.x + yERROR: error in method definition: function Bar2.foo must be explicitly imported to be extended

Where does imported variable points to?

The import case

  • x imported to the current module (@__MODULE__) points to Bar3.x.
  • You cannot assign x in the current module.
module Bar3
    x = 1
    foo() = x
    export x
    export foo
end
Main.Bar3
julia> import .Bar3: foo, x
julia> foo(y) = x + yfoo (generic function with 2 methods)
julia> x = 9ERROR: cannot assign a value to imported variable Bar3.x from module Main
julia> foo(3)4
julia> Bar3.x = 333333
julia> x333
julia> foo(3)336
julia> Bar3.x === xtrue

The using case

module Bar4
    x = 1
    foo() = x
    export x
    export foo
end
Main.Bar4
julia> using .Bar4
julia> Bar4.foo(y) = x + y
julia> x = 9.129.12
julia> Bar4.x === xfalse
julia> foo(3)12.12

The behavior is strange, that the first call of x does matter.

julia> module Bar4
           x = 1
           foo() = x
           export x
           export foo
       endMain.Bar4
julia> using .Bar4
julia> x1
julia> Bar4.x === xtrue
julia> x = 9.12ERROR: cannot assign a value to imported variable Bar4.x from module Main
